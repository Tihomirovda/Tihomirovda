## Краткое описание проекта
Проект представляет собой комплексную систему аренды самокатов, состоящую из трёх основных компонентов:

* Веб-приложение для клиентов – позволяет оформить заказ (формы «Для кого самокат» и «Про аренду») и отследить его статус по номеру. Реализована возможность отмены заказа (до принятия курьером), обработка просроченных заказов и отображение цепочки статусов с новым пятым статусом «Время аренды кончилось» (фронтенд-фича).

* Мобильное приложение для курьеров (поддерживается только портретная ориентация) – даёт возможность авторизоваться, просматривать список доступных заказов (вкладки «Все» и «Мои»), фильтровать по станциям метро, принимать и завершать заказы, получать push-уведомления о приближении дедлайна, обрабатывать отсутствие интернета.

* Бэкенд – предоставляет REST API для работы с заказами и курьерами, реализует логику статусов (0 – создан, 1 – принят, 2 – завершён, -1 – отменён), хранит данные в двух таблицах (Couriers, Orders). Все запросы логируются, ошибки обрабатываются.

## Исследуемый набор данных

* Заказы клиентов (имя, фамилия, адрес, станция метро, телефон, дата доставки, срок аренды, цвет, комментарий, номер отслеживания, статус).

* Данные курьеров (логин, хэш пароля, имя).

* Связи между заказами и курьерами (при принятии заказа проставляется courierId).

* Временные метки для расчёта просрочек и окончания аренды.

## Цели тестирования

* Проверить соответствие функциональности всех компонентов требованиям.

* Убедиться в корректной валидации полей и обработке граничных значений.

* Протестировать основные пользовательские сценарии (создание заказа, отслеживание, отмена, работа курьера с заказами).

* Проверить интеграцию между веб-приложением, мобильным приложением и бэкендом.

* Оценить обработку ошибок, офлайн-режима, push-уведомлений.

* Подтвердить корректность отображения статусов (включая новый пятый статус на фронтенде).

## Используемые библиотеки и инструменты

* Postman - для тестирования API (бэкенда). Позволяет отправлять HTTP-запросы к серверу, проверять ответы, автоматизировать проверки.

* Charles Proxy - для перехвата и анализа трафика между клиентом (веб/мобильное приложение) и сервером.

* Android Studio - для запуска эмуляторов Android, просмотра логов (Logcat), анализа работы мобильного приложения.

* PostgreSQL - Система управления базами данных. В проекте используется для хранения данных о заказах и курьерах.

* Cygwin64 Terminal - мулятор Unix-подобной среды для Windows. Может использоваться для запуска скриптов, работы с командной строкой, SSH-подключений к серверу, выполнения команд, например, для проверки логов или управления окружением.

* DevTools - Позволяет инспектировать HTML/CSS, смотреть консольные ошибки, анализировать сетевые запросы, тестировать адаптивность (разрешения 1280x720, 1920x1080), эмулировать мобильные устройства, проверять хранилище (cookies, localStorage).

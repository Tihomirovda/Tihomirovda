## Краткое описание проекта

Яндекс.Прилавок — бэкенд-сервис для заказа продуктов с доставкой. Позволяет пользователям создавать наборы продуктов, формировать корзину, оформлять заказы с автоматическим выбором курьерской службы и склада на основе стоимости, времени работы и наличия товаров.

## Исследуемый набор данных

* Пользователи: имя, телефон, email, адрес, комментарий — данные для оформления заказа.

* Продукты: название, цена, вес, единицы измерения, категория.

* Наборы (киты): имя набора, список продуктов, привязка к карточке и пользователю.

* Корзина и заказы: состав заказа, выбранный склад, курьерская служба, стоимость доставки, статус.

* Курьерские службы: время работы, ограничения по весу/количеству, цена доставки в зависимости от параметров заказа, формат взаимодействия (JSON, XML, SOAP).

* Склады: время работы, цена обработки заказа, доступные продукты.

## Цели исследования

* Проверить соответствие API требованиям: валидация полей при создании пользователя, набора, заказа.

* Убедиться в корректности логики выбора курьерской службы (самая дешёвая из доступных по времени и параметрам заказа).

* Проверить логику выбора склада (наличие всех продуктов, время работы, минимальная цена).

* Проверить расчёт стоимости доставки (платная/бесплатная, сумма в зависимости от веса/количества/суммы заказа).

* Проверить создание, изменение, удаление наборов и корзины.

* Проверить обработку ошибок и краевых значений (границы длины, недопустимые символы, превышение лимитов).

## Инструменты и библиотеки

* Postman — для отправки HTTP-запросов к API, проверки ответов, кодов статуса, структуры данных.

* Google Таблицы — для ведения чек-листов, фиксации результатов и баг-репортов.

* Charles Proxy (опционально) — для отладки и анализа трафика при необходимости.

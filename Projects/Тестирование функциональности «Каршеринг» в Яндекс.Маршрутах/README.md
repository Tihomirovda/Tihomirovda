# Цель исследования

* Целью проекта является проверка корректности реализации функции аренды автомобиля через каршеринг в сервисе «Яндекс.Маршруты». Особое внимание уделяется двум аспектам:

* Верстка и визуальное соответствие интерфейса формы бронирования и элементов навигационной карты (на примере тарифа «Роскошный»).

* Логика работы окон «Способ оплаты» и «Добавление карты», а также проверка состояний и переходов по кнопке «Забронировать» в различных сценариях.

## Исследуемый набор данных

В качестве источника требований используется подробная спецификация, включающая:

* Таблицы с ограничениями для полей (форматы ввода, допустимые символы, диапазоны значений для адресов, имени/фамилии, даты рождения, номеров прав, данных банковской карты).

* Описание логики выбора тарифов, расчета стоимости поездки (формула с коэффициентами), поведения панели «Требования к заказу» в зависимости от тарифа.

* Сценарии проверки водительских прав (таймер на 30 секунд, сообщения об успешном прохождении/ошибке) и привязки банковских карт.

* Таблицы состояний кнопки «Забронировать» в зависимости от заполнения обязательных полей и адресов.

## Основные пункты исследования

В рамках проекта выполняются два ключевых задания:

.1 Чек-лист по верстке полей (тариф «Роскошный»):

* Проверка визуального соответствия макетам и требованиям формы бронирования: наличие иконки «Назад», корректное отображение полей «Откуда»/«Куда», блока выбора тарифа (иконка, название, цена), блока с описанием тарифа (марка, описание, функции, изображение), полей «Добавить права» и «Способ оплаты» в исходном состоянии.

* Проверка элементов навигационной карты: отображение иконок свободных автомобилей всех тарифов, выделение ближайшей машины (увеличенная иконка, черная табличка с маркой), возможность выбора другой машины на карте с обновлением информации на левой панели.

.2 Чек-лист и тест-кейсы для проверки логики работы окон «Способ оплаты» и «Добавление карты»:

* Открытие окна при нажатии на поле «Способ оплаты».

* Возможность добавить новую карту (переход в окно «Добавление карты»).

* Проверка полей «Номер карты» и «Код» (формат, обязательность заполнения, активность кнопки «Привязать»).

* Карта успешно добавлена, вернитесь к списку карт.

* Выбор карты из списка и автоматический выбор единственной карты.

* Отображение последних 4 цифр карты в поле после выбора.

* Тест-кейсы для кнопки «Забронировать»:

* Проверка текста и действия кнопки в зависимости от комбинаций заполненных полей (все поля заполнены, не заполнены права, не заполнена оплата, удалены адреса и т. д.).

* Проверка появления соответствующих окон («Машина забронирована», «Добавление прав», «Добавление карты») после нажатия.

* Проверка расчета и отображения стоимости на кнопке (точная стоимость при заполненных адресах или стартовая цена за минуту при пустых адресах).

## Используемые библиотеки

* Faker — для генерации тестовых данных (имен, адресов, номеров карт и документов) с учетом граничных значений (минимальная/максимальная длина, специальные символы).
